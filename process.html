COM216 Website Development Process

A comprehensive, chronological documentation of how The Story of Information & The Telegraph was built.

Project Summary

This page is a step-by-step account of our collaborative process: initial concept, iterative HTML drafts, design modernization, adding JavaScript interactivity, and structuring the final multi-page site for deployment. Below, you will find the key prompts that guided each step, commentary on what changed, and interactive code snippets that you can copy and reuse.

Phases of Development

Phase 1: Foundation & Merging

Prompt: "Merge this and this into one code."

We started with two separate HTML files—a narrative page and a telegraph simulator. The first step was to carefully merge them into a single, functional file. This involved scoping the unique CSS of each component to prevent conflicts and ensure both parts rendered correctly on one page, establishing the project's foundation.

Phase 2: Polish & Modernization

Prompt: "Let's polish this... make the color, design, and font more modern, but keep everything I did original."

With the core content merged, we focused on user experience and visual design. We implemented a sleek, dark-mode theme, updated the typography to a clean sans-serif font (Inter), and added subtle interactive effects. The goal was to create a professional feel while preserving the unique, vintage aesthetic of the Telegraph Simulator as a self-contained element.

Phase 3: Content Expansion & Organization

Prompt: "Can we add more class information that we have talked about... and make sure everything is in order by chapter please."

We expanded the site's narrative by adding new sections based on your class material (such as the summary of Chapter 4 on Babbage). We then reorganized all the content to follow the chronological order of the source material, creating a more logical and historically accurate flow for the user to follow.

Phase 4: Architectural Overhaul (Multi-Page Site)

Prompt: "I would like to set up my website... not just using one huge HTML but each page have their own different one that can be navigated."

This was a crucial architectural decision. We converted the single-file website into a traditional multi-page site by creating separate, interlinked HTML files for each section. This makes the project more scalable, easier to manage, and aligns with standard web development practices, especially for hosting on platforms like GitHub Pages.

Phase 5: Documentation & Final Polish

Prompt: "Re-code everything with your ideas... I want someone to be able to see what exactly I did and the process and be able to repurpose it."

The final step was to create this very page. We added smooth scrolling for better navigation, interactive "Copy" buttons for the code snippets to make them reusable, and expanded the commentary to ensure the development process is transparent and easy to understand.

/* Smooth scrolling CSS */
html {
    scroll-behavior: smooth;
}

/* Copy button JavaScript */
document.querySelectorAll('.copy-btn').forEach(button => {
    button.addEventListener('click', () => {
        const pre = button.nextElementSibling;
        const code = pre.querySelector('code');
        navigator.clipboard.writeText(code.innerText).then(() => {
            button.innerText = 'Copied!';
            setTimeout(() => { button.innerText = 'Copy'; }, 2000);
        });
    });
});

Final Reflection

This project's development highlights the value of an iterative process. We started with a simple functional goal (merging files) and progressively added layers of design, interactivity, and content. Each phase built upon the last, allowing us to refine ideas and adapt the project's structure—transforming it from a single scrolling page into a complete, multi-page website.

Technically, the project demonstrates effective use of vanilla JavaScript for DOM manipulation, event handling, and creating interactive features like popups, a quiz, and an audio simulator without relying on external frameworks. Creatively, it shows how to blend different thematic styles into a single, cohesive user experience.
